# Настройка Java 25 для проекта

## Проблема
Maven использует JDK 21 вместо JDK 25, что вызывает ошибку компиляции:
```
error: release version 25 not supported
```

## Решение

### Вариант 1: Установить JAVA_HOME через графический интерфейс

1. Откройте **Панель управления** → **Система** → **Дополнительные параметры системы**
2. Нажмите **Переменные среды**
3. В разделе **Системные переменные** найдите `JAVA_HOME`
4. Если переменная существует, измените значение на: `C:\Program Files\Java\jdk-25`
5. Если переменной нет, создайте новую:
   - Нажмите **Создать**
   - Имя переменной: `JAVA_HOME`
   - Значение переменной: `C:\Program Files\Java\jdk-25`
6. Нажмите **ОК** во всех окнах
7. Перезапустите терминал/PowerShell

### Вариант 2: Установить JAVA_HOME через PowerShell (от имени администратора)

```powershell
[System.Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk-25", "Machine")
```

После выполнения перезапустите терминал.

### Вариант 3: Временная установка для текущей сессии

Если нужно только для текущей сессии PowerShell:

```powershell
$env:JAVA_HOME = "C:\Program Files\Java\jdk-25"
```

## Проверка

После настройки проверьте:

```powershell
# Проверка JAVA_HOME
$env:JAVA_HOME

# Проверка версии Java
java -version

# Проверка версии, используемой Maven
mvn -version
```

Должно показать:
- `JAVA_HOME`: `C:\Program Files\Java\jdk-25`
- `java -version`: `java version "25.0.1"`
- `mvn -version`: `Java version: 25.0.1`

## Сборка проекта

После настройки JAVA_HOME:

```bash
mvn clean install
```

## Примечания

- Убедитесь, что JDK 25 установлен по пути `C:\Program Files\Java\jdk-25`
- Если JDK 25 установлен в другом месте, используйте соответствующий путь
- После изменения системных переменных необходимо перезапустить терминал/IDE
